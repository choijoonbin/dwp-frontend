# BE → FE API 결과·완료 문서 (우리 api-spec)

> **목적**: 백엔드가 **작업 완료 후** 업로드한 결과·완료 문서를 보관합니다.  
> **최종 업데이트**: 2026-01-26

---

## 1. 폴더 역할

- **백엔드**가 작업을 완료한 뒤, **결과·답변·완료 문서**를 이 폴더(`docs/api-spec/`)에 업로드(PR, 공유 등)합니다.
- **프론트**는 이 폴더에서 결과 문서를 확인한 후 FE 개발을 진행합니다.

---

## 2. 규칙

| 구분 | 내용 |
|------|------|
| **업로드** | 백엔드: 작업 완료 후 `{요청문서제목}_result.md` 형식으로 결과 문서 업로드 (반복 시 `_result_v1.md` 등) |
| **확인** | 프론트: `docs/api-spec/`에서 `*_result*.md` 확인 후 작업 |
| **추가 질문** | 프론트: 추가 질문이 있으면 **요청 문서**를 `_v1`, `_v2`로 갱신해 `docs/backend-src/docs/api-spec/`에 반영 |

---

## 3. FE 요청 문서 위치

- 프론트가 백엔드에 보내는 **요청·검토 문서**는 **백엔드의 `docs/api-spec/`** 에 둡니다.
- 프론트 저장소 기준: `docs/backend-src/docs/api-spec/` (backend-src는 백엔드 프로젝트 링크)
